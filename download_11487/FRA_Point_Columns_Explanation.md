Column content explanation

**Change Status**
Describes current status of each FRA point record versus previous edition of the file.
   NEW : new record (record also highlighted in GREEN)
   DEL   : withdrawn record (record also highlighted in RED)
   AMD   : changed record (change also highlighted in BLUE)
If there is no change from previous edition, the relevant box remains empty.
In the next FRA points file, the row marked as DEL will be removed from the file.
**Point Type**
Describes the NAVAID type when used as FRA point.  It contains relevant abbreviation indicating the type as follows:
   VOR : VOR NAVAID
   DME : VOR/DME NAVAID
   VORDME : VOR/DME NAVAID
   VORTAC : VORTAC NAVAID
   NDB : NDB NAVAID
   LOCATOR : Locator or Marker NAVAID
   NDBDME : Collocated NDB and DME NAVAID
If it refers to the 5LNC the relevant box is empty.
**FRA Point**
Describes the FRA point name.  It contains either two or three letter abbreviation of relevant NAVAID or five-letter name code (5LNC) of relevant FRA point (NM or LNO or DIMLO).
**FRA Point Latitude**
Describes the FRA point Latitude. Format is XDDMMSS, X being N or S.
**FRA Point Longitude**
Describes the FRA point Longitude. Format is XDDDMMSS, X being E or W.
**FRA Name”
Describes the name of the FRA area to which relevant FRA point belongs.  It contains abbreviated project name (SEENFRA).
**FRA Status En-Route**
Describes the en-route status of the relevant FRA point.  It contains FRA point coded letter:
   EX : Combined Entry/Exit
   E   : Entry
   X   : Exit
   I    : Intermediate
If there is no such status the relevant box remains empty, expressed by (-).
**FRA Status ARR/DEP**
Describes the arrival and/or departure status of the relevant FRA point.  It contains FRA point coded letter:
   AD : Combined Arrival/Departure
   A   : Arrival
   D   : Departure
If there is no such status the relevant box remains empty, expressed by (-).
**Arrival Airport(s)**
Describes FRA point relevant arrival airport/s.  It contains the four-letter ICAO location indicator of the relevant arrival aerodrome/s in accordance with Doc 7910.
**Departure Airport(s)**
Describes FRA point relevant departure airport/s.  It contains the four-letter ICAO location indicator of the relevant departure aerodrome/s in accordance with Doc 7910.
**FLOS**
Describes the Flight Level Orientation Scheme (FLOS) over relevant FRA point.  In contains the direction of cruising levels used expressed by the following terms:
   ODD : ODD (Entry or Exit)
   EVEN : EVEN (Entry or Exit)
   ALL : ODD and EVEN (Entry or Exit)
   ODD/EVEN : ODD (Entry) and EVEN (Exit)
   EVEN/ODD : EVEN (Entry) and ODD (Exit)
  ALL/ALL : ODD and EVEN for both Entry and Exit
  *: See Remarks column
  If FLOS is unknown for the point - not published in AIP the relevant box remains empty, expressed by (-).
**Level Availability**
Describes the vertical availability of the relevant FRA point.  It contains the expression in accordance with ICAO (GND, 9500FT, FL095, etc.) of relevant lower and upper limits.
**Time Availability**
Describes the timely availability of the relevant FRA point.  It contains the time availability expression in accordance with “NM Flight Planning Requirements - Guidelines”.
**Airspace Location Indicators**
Describes the airspace (FIR or ACC) to which  the relevant FRA point belongs.  It contains the four-letter ICAO location indicator of the relevant airspaces in accordance with Doc 7910. ACC takes precedence over FIR.
**Cross-Border FRA States**
Describes the State/s of the relevant FRA point.  It contains the name of the State or States in which this FRA point is located within the same FRA airspace.
**Remarks**
Describes any unclear way of publication in AIP or published through another mean (e.g. NOTAM).  It contains the plain text related to the relevant FRA point.
